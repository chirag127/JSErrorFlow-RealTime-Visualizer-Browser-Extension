Privacy Policy

Last Updated: April 14, 2025

Introduction

This Privacy Policy explains what information JavaScript Error Visualizer collects, how it is used, and how it is protected. We are committed to ensuring the privacy and security of your data while providing a valuable debugging tool for web developers.

JavaScript Error Visualizer is designed with privacy in mind, processing all data locally within your browser and never transmitting your information to external servers.

Required Permissions

JavaScript Error Visualizer requires the following permissions to function properly:

storage

This permission allows the extension to save your preferences and settings, such as highlight styles, domain allowlists/blocklists, and other configuration options. All settings are stored locally in your browser using the chrome.storage API.

scripting

This permission allows the extension to inject content scripts into web pages to capture JavaScript errors and highlight associated DOM elements. This is essential for the core functionality of the extension.

tabs

This permission allows the extension to access information about the currently active tab, such as its URL. This is necessary to associate errors with specific tabs and to determine if the extension should be enabled for the current domain.

host_permissions (<all_urls>)

This permission allows the extension to run on all websites you visit. This is necessary to capture JavaScript errors on any website. You can restrict this permission through the extension's domain settings, enabling it only for specific websites.

Information We Collect

JavaScript Error Visualizer collects the following information locally on your device:

  • JavaScript Error Data: Error messages, stack traces, source code locations (file names, line numbers, column numbers), and associated DOM elements.
  • Source Maps: The extension may fetch and process JavaScript source maps from websites to translate minified code back to its original form for better error reporting.
  • User Settings: Your configuration preferences, such as highlight styles, domain allowlists/blocklists, and other extension settings.

What We DO NOT Collect

JavaScript Error Visualizer does not collect or transmit:

  • Personal information
  • Browsing history
  • Login credentials
  • Any data from form inputs
  • Any error data or source code snippets to external servers

How We Use Your Information

All information collected by JavaScript Error Visualizer is:

  • Processed Locally: All data processing occurs entirely within your browser.
  • Stored Locally: All data is stored locally on your device using browser storage APIs (chrome.storage).
  • Never Transmitted: No error data, source code snippets, or website content is ever transmitted to external servers.

The collected information is used solely to provide the core functionality of the extension:

  • Capturing and displaying JavaScript errors
  • Highlighting associated DOM elements
  • Providing detailed error information
  • Applying your preferred settings and configurations

Data Storage & Security

JavaScript Error Visualizer stores data in the following ways:

  • Error Data: Stored temporarily in memory and/or local browser storage. This data is cleared when you close the tab, reload the page, or manually clear errors through the extension interface (depending on your settings).
  • User Settings: Stored in the browser's local storage (chrome.storage.sync or chrome.storage.local) and persists between browser sessions.

Since all data is processed and stored locally on your device, the security of your data depends on the security of your browser and operating system. The extension does not introduce any additional security risks beyond those inherent to your browser.

Third-Party Services

JavaScript Error Visualizer does not use any third-party services, analytics, or tracking tools. All functionality is self-contained within the extension.

The only external resources the extension may access are:

  • JavaScript source maps hosted on the websites you visit (to improve error reporting)

These resources are accessed directly by your browser and are not shared with any third parties.

User Rights

As a user of JavaScript Error Visualizer, you have the right to:

  • Access: View all data stored by the extension in your browser's storage.
  • Modify: Change your settings and preferences at any time through the extension's settings page.
  • Delete: Clear all stored data by clearing the extension's storage or uninstalling the extension.
  • Restrict: Limit the extension's access to specific domains through the domain settings.

To exercise these rights, you can use the extension's built-in settings and controls or your browser's extension management features.

Changes to This Privacy Policy

We may update our Privacy Policy from time to time to reflect changes in our practices or for other operational, legal, or regulatory reasons.

If we make significant changes to this Privacy Policy, we will notify you by:

  • Updating the "Last Updated" date at the top of this Privacy Policy
  • Providing a notification within the extension
  • Posting a notice on our GitHub repository

We encourage you to review this Privacy Policy periodically to stay informed about how we are protecting your information.

Contact Information

If you have any questions or concerns about this Privacy Policy or JavaScript Error Visualizer's privacy practices, please contact us by:
